# Change Documentation

Every meaningful change should add one Markdown file in this directory.

## File Naming

Use this format:

```text
YYYY-MM-DD-short-change-name.md
```

## Required Sections

- `## Summary`
- `## Verification`
- `## Context Updates`

## Agent Rule

When an agent changes automation or documentation in this repo, it must either add a change document or explain in the PR why the change is purely mechanical.
